# 🏥 各組別績效管理系統 (Multi-Group Decision Intelligence) 專案紀錄與指南

## 一、 修改歷程與編寫紀錄 (Changelog)

本階段的核心改造目標是將原本寫死欄位的「神復組」系統，升級為可容納全院 **10 個大組別**、且各組轄下**科別數量動態擴充**的通用架構。

### 1. 資料庫層級 (Supabase)
*   **動態指標表 (`daily_metrics_v2`)**：
    *   捨棄傳統 SQL 寫死欄位（如 `careCountNeuro`），改用 **PostgreSQL JSONB** 格式。
    *   資料結構改為 `care_counts: { "dept_id": count }`，達成無需修改資料表即可支援無限科別。
*   **人員名單隔離模式**：
    *   在 `staff` 資料表新增 `group_id` 欄位。
    *   實作人員複製腳本，將初始名單同步至 10 個組別中，達成各組人員獨立增刪，互不連動。

### 2. 前端應用層 (React + TypeScript)
*   **設定驅動架構**：
    *   新增 `src/config/groups.ts` 為系統核心。透過此設定檔即可定義組別名稱及其對應的科別清單。
*   **動態 UI 渲染引擎**：
    *   **CareEntry / NewPatientEntry**：利用動態 Map 機制取代靜態元件。系統會根據當前組別與月份，自動計算並長出正確數量的輸入框。
    *   **Dashboard**：重構 CMI 設定區域與圖表組件，使其能動態顯示該組別的權重設定與占比圓餅圖。
*   **路由機制**：
    *   開發 `GroupSelector` 首頁，作為進入各個功能模組的入口。

### 3. 管理員驗證與年度擴充 (2026-04-25)
*   **管理員驗證強化**：
    *   預設密碼變更為 `A200`。
    *   登入 Modal 移除密碼提示文字，提升安全性。
*   **年度選擇動態化**：
    *   年度生成由靜態範圍改為動態計算（起始 2026，結束為當前年份 + 1）。
    *   目前選單已自動支援至 2028 年，且未來無需手動修改程式碼即可自動延展。

## 三、 系統使用說明書 (User Manual)

### 1. 登入與組別選擇
*   開啟系統網址後，首頁會呈現 10 個核心組別（如：心腎感染組、重症 ICU 等）。
*   點選對應方塊即可進入該組別的填報畫面。

### 2. 每日照護數填報 (Care Entry)
*   左側側邊欄選擇「**照護數填寫**」。
*   上方切換「人員」與「月份」。
*   表格中會顯示該組別所有的細分科別，點擊輸入框填入當日數值，**系統會自動即時儲存至雲端**。

### 3. 新接病人填報 (New Patient Entry)
*   左側側邊欄選擇「**新接數與病歷號碼**」。
*   除各科別新接數外，亦提供「**其他**」欄位供填報。
*   右側可點擊「新增病歷號」記錄具體號碼。
*   若「新接總數」與「填寫的病歷號數量」不符，系統會出現 **黃色警告標誌** 提醒檢查。

### 4. 統計總覽 (Dashboard) - *管理員功能*
*   進入時需輸入管理員密碼 (預設: `A200`)。
*   **CMI 權重設定**：頁面最上方可針對該組別各科別設定權重，數值會即時影響下方所有得分計算。
*   **視覺化圖表**：提供全組排名、科別占比圓餅圖以及個人趨勢分析。
*   **匯出資料**：點擊「匯出 Excel」可下載目前的績效統計月報。

### 5. 人員名單管理 (Staff Management)
*   在側邊欄最下方點擊「人員名單管理」（需管理員密碼）。
*   可在「該組別」內新增或移除員工。此動作目前已隔離，僅會影響當前組別的名單。
